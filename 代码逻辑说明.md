# FCN Watchlist 代码逻辑说明

写给接手维护的人。看完这份你能明白：数据从哪来、怎么变成网站、每个文件负责什么、
想改东西该改哪里。不需要会写 Python 也能看懂大流程。

---

## 一、整体架构（一句话）

```
3 个 Excel（你每周换）
      │
      ▼
generate_watchlist.py  ──①读Excel ──②拉行情(yfinance) ──③拉分析师/财报(富途) ──④AI写分析(DeepSeek) ──⑤拉市场热点 + 板块热力图
      │
      ▼
watchlist.json   ← 网站的【唯一数据源】
      │
      ▼
index.html （纯前端，fetch 读 watchlist.json 渲染页面）
      │
      ▼  git push
GitHub Pages → 公网网站
```

**关键心智模型：**
- 网站本身（`index.html`）是「死」的，它只负责把 `watchlist.json` 画出来。
- 所有内容、所有更新，本质都是在**重新生成 `watchlist.json`**。
- 上线 = 把新的 `watchlist.json` `git push` 上去，GitHub Pages 自动重建。

## 二、文件职责

| 文件 | 职责 |
|------|------|
| `generate_watchlist.py` | **主程序 / 全部核心逻辑都在这**。读 Excel、拉数据、调 AI、写 `watchlist.json` |
| `repair_watchlist.py` | 复用主程序的函数，只针对「分析不完整」的个别标的重跑一遍（batch=1，避免连坐） |
| `backfill_price_history.py` | 一次性工具：只回填价格历史（2 年日线），不重跑 AI |
| `index.html` | 前端单页。含一个 ADMIN 后台（密码 `sp2026`）可手改数据并导出 `watchlist.json` |
| `watchlist.json` | 程序的产物 + 网站的输入。手不要直接编辑（除非很清楚结构） |
| 3 个 `.xlsx` | 程序的输入。每周替换 |

## 三、generate_watchlist.py 的 5 个阶段

主入口是 `main()`（文件末尾）。运行时控制台会打印 `[1/4] [2/4] ...` 对应下面阶段。

### ① 读 Excel（`_parse_excel` / `_parse_terms_excel`）
- 默认「三类型模式」：分别读 `低波精选组 / 高波精选组 / 市场热度榜` 三个 Excel，每个取前 `TOP_N_PER_TYPE`（=10，可命令行 `--top` 改）只。（内部类型键仍为 稳健/进取/热度，只是文件改了名）
  > 注：实际每类放约 20 只时，把 `TOP_N_PER_TYPE` 调大或用 `--top 20`。当前默认值是 10。
- 同一标的出现在多个 Excel → 合并，`types` 字段记多个标签，不重复。
- 代码归一：港股补成富途 5 位码（`HK.00700`），美股补 `US.NVDA`；同时算出
  - `display_code`（页面显示，如 `NVDA.US` / `700.HK`）
  - `yf_code`（喂给 yfinance，如 `NVDA` / `0700.HK`）
- Excel 里的条款（执行价/敲入价/敲出价/票息）是**小数**，`<2` 的自动 ×100 变百分比。

### ② 拉行情（yfinance，`get_sparkline` / `get_price_history` / `get_realtime_context`）
- 每只标的拉 2 年日线收盘价（画详情页走势图）+ 10 周迷你 sparkline + 最新涨跌幅。
- `initialPrice`（期初观察价）= **最近一个已完结交易日**的收盘价；当天未收盘的「活跃 bar」会被 `_drop_live_bar` 丢掉，避免用到盘中价。
- 同时抓 Yahoo 的公司简介 + 最近新闻，作为后面喂给 AI 的背景材料。

### ③ 拉分析师/财报（富途 OpenAPI，`get_futu_context`）
- 通过本机的 **Futu OpenD**（需登录）拉：分析师一致评级 / 目标价、近期评级变动、利润表 / 现金流 / 分业务营收 / 业绩 beat-miss。
- **OpenD 没开也不报错**，函数返回一句「连接失败」，AI 分析照常生成，只是少这块输入。

### ④ AI 写分析（DeepSeek，`analyze_batch`）
- key 来源优先级：环境变量 `DEEPSEEK_API_KEY` > 本目录 `deepseek_key.txt`。
- 把②③的行情 + 财报 + 新闻拼成上下文，连同 `SYSTEM_PROMPT` / `BATCH_PROMPT` 一起发给 DeepSeek，
  每 `BATCH_SIZE`（=3）只一批，`MAX_WORKERS`（=3）个线程并发。
- 模型按固定 schema 返回结构化分析（`tagline / intro / B / C / health_conc / c4_prose / risks / bullets`）。
- 生成后 `_validate_ana` 校验字段是否齐全；**不完整的标的会自动单股重试最多 3 次**。
- ⚠️ Prompt 里硬性要求：正文不得出现「富途 / Yahoo」等数据平台名，一律归因到「分析师共识 / 公司财报 / 交易所」等底层出处。

### ⑤ 市场热点 + 板块热力图（首页用）
- `get_market_hotspot`：用 **Google News RSS**（免费、免 key）抓美股 + 港股新闻 → DeepSeek 综述成「本周热点」。
- `get_sector_heatmap`：用富途行业板块数据算各板块过去一周涨跌，做成首页热力图。
- 这两个**生成失败会自动沿用上一版**（`watchlist.json` 里的旧值），保证首页不留白。
- **财经日历**（`meta.calendar`）脚本不自动抓（investing.com 反爬），靠每周人工/WebFetch 写入，脚本只负责「不覆盖」。

### 最后：合并写盘
所有标的 + `meta`（周次、精选 ID、热点、热力图、日历）打包写进 `watchlist.json`，
并在控制台提示下一步 `git add/commit/push`。

## 四、watchlist.json 结构速览

```jsonc
{
  "_generated": { "by", "model", "at", "count", "excel" },   // 生成元信息
  "meta": {
    "week", "publishDate", "nextUpdate",
    "featuredIds": [...],        // 本周精选（每类型自动选 2 只）
    "marketHotspot": {...},      // 首页本周热点
    "sectorHeatmap": [...],      // 首页板块热力图
    "calendar": {...}            // 财经日历（人工维护）
  },
  "stocks": [
    {
      "code", "name", "industry", "market", "types": ["稳健"],  // types 为内部键：稳健/进取/热度，网站分别显示为 低波精选/高波精选/市场热度型（勿改内部键）
      "price", "marketCap", "currency", "priceChange",
      "sparkline": [...], "priceHistory": {dates, closes},
      "coupon","strike","ki","ko","kiType","tenor","risk",  // FCN 条款
      "bullets": [...],           // 卡片要点
      "report": { tagline,intro,B,C,health_conc,c4_prose,risks },  // 详情页分析
      ...
    }
  ]
}
```

## 五、想改某样东西，改哪里？

| 想改 | 改哪 |
|------|------|
| 每类取多少只 | `generate_watchlist.py` 顶部 `TOP_N_PER_TYPE`，或运行加 `--top 20` |
| AI 用的模型 / API 地址 | 顶部 `MODEL`、`API_BASE` |
| 没填条款时按 IV 估的默认条款 | `calc_fcn_terms()` 函数 |
| AI 分析的写法 / 口吻 / 结构 | `SYSTEM_PROMPT` / `BATCH_PROMPT`（约 388 行起） |
| 首页热点新闻的关键词 | `_fetch_us_news` / `_fetch_hk_news` 里的查询词 |
| 网站长相 / 配色 / 布局 | `index.html`（前端） |
| 后台密码 | `index.html` 里搜 `ADMIN_PASSWORD`（当前 `sp2026`） |

## 六、依赖

`requirements.txt`：`pandas openpyxl`（读 Excel）、`yfinance`（行情）、`openai`（调 DeepSeek，兼容 OpenAI SDK）、
`json_repair`（修 AI 返回的残缺 JSON）、`futu-api`（富途）、`feedparser`（Google News RSS）。

> 小提示：`index.html` 里 ADMIN 帮助文字写的是「Claude API」，实际用的是 **DeepSeek**（见 `MODEL`）。只是文案，不影响运行。
