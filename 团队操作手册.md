# FCN Watchlist 网站更新手册（团队版）

网站地址：https://daveliao999.github.io
仓库地址：https://github.com/daveliao999/.github.io

整个网站的内容由 `watchlist.json` 驱动，由 `generate_watchlist.py` 根据三个 Excel 自动生成。
**每周更新 = 替换三个 Excel → 双击「更新网站.bat」→ 等约 25 分钟 → 自动上线。**

---

## 一、首次配置（每人只需做一次，约 20 分钟）

### 1. 获得仓库权限
把你的 GitHub 账号发给 Dave，由他在仓库 Settings → Collaborators 中添加。
收到邀请邮件后点 Accept。

### 2. 安装软件
- **Python 3.10+**：https://www.python.org/downloads/ ，安装时务必勾选 **"Add Python to PATH"**
- **Git**：https://git-scm.com/download/win ，一路默认下一步即可

### 3. 克隆仓库
打开「命令提示符」（Win 键搜 cmd），逐行执行：

```
cd %USERPROFILE%\Desktop
git clone https://github.com/daveliao999/.github.io.git fcn-watchlist
cd fcn-watchlist
pip install -r requirements.txt
```

首次 push 时 Git 会弹出 GitHub 登录窗口，用自己的账号登录一次即可（之后自动记住）。

### 4. 配置 DeepSeek key
向 Dave 索取 API key，在 `fcn-watchlist` 文件夹里新建文本文件 **`deepseek_key.txt`**，
把 key 粘贴进去保存（只有一行，没有引号没有空格）。

> ⚠️ 这个 key 文件已被 .gitignore 排除，**绝对不要**把 key 写进任何会提交的文件——本仓库是公开的。

> 首页「本周热点分析」的美股与港股新闻都来自 Google News（免费、免 key），无需额外配置。

### 5.（可选，推荐）安装富途 OpenD
分析报告里的"分析师评级/目标价/富途财报"数据来自富途 OpenAPI。
没有它脚本也能正常跑完，只是分析少这一个维度。

- 下载安装 Futu OpenD（图形界面版）：https://www.futunn.com/download/OpenAPI
- 用**自己的**富途账号登录（不要共用账号，会把对方挤下线）
- 跑脚本前保持 OpenD 开着并已登录即可

## 二、每周更新流程（约 30 分钟，其中 25 分钟是等待）

1. **更新三个 Excel**（在 `fcn-watchlist` 文件夹内，文件名不能改）：
   - `低波精选组.xlsx` — 低波精选 20 只
   - `高波精选组.xlsx` — 高波精选 20 只
   - `市场热度榜.xlsx` — 市场热度型 20 只

   列格式保持不变：区域 / 代码（`NVDA US` / `9988 HK`，或纯代码 NVDA/9988 均可，市场以「区域」列为准）/ 名称 / 行业 / 执行价 / 敲入价 / 敲入类型 / 敲出价 / 票息（小数，0.8 = 80%）。
   同一标的出现在多个 Excel = 自动合并为多类型标签。

2. **（可选）启动并登录富途 OpenD**

3. **双击 `更新网站.bat`**，等待窗口显示 ✅ 完成。
   窗口会依次：拉取最新代码 → 生成分析（约 25 分钟）→ 提交 → 推送上线。

4. 几分钟后打开 https://daveliao999.github.io 检查：
   - 三个策略卡片各 20 只标的
   - 抽查两三只标的的详情页分析是否完整

## 三、执行价/敲入价 AI 建议（询价前的前置步骤）

在填「执行价/敲入价」之前，先按 IV 自动反解一版结构，再拿去交易台询价。全程三个脚本，依次跑：

1. **名单** `lists_input.xlsx`：至少两列 **组别 / 代码**（IV 列留空，下一步自动填）。
   - 组别（低波/高波/热度）只用于组织名单；**执行价只看 IV 水平，与在哪个榜单无关**——
     同一标的进多个榜单，执行价/敲入价都一样。
2. **拉 IV**：`python fetch_iv.py` → 用 OpenD 取每只「90% 现价档、6 个月 put」的 IV，
   写回 `lists_input.xlsx` 的 IV 列（港股无期权自动退回 6M HV）。**需 OpenD 开着并登录**。
3. **反解结构**：`python strike_advisor_pipeline.py` → 输出 `建议结构.csv`
   （每标的一行，执行价/敲入价为整数；规则见下框）。
4. **填进榜单 Excel**：`python fill_terms.py` → 把执行价/敲入价按代码写进三个榜单 Excel
   （`低波精选组 / 高波精选组 / 市场热度榜`），并把代码统一成 `NVDA US / 9988 HK`；票息列留空。
5. **询价 → 生成**：把三个 Excel 发交易台 → 真实票息填回票息列 → 走「二、每周更新流程」推网站。
   - **改了规则或重拉 IV 后**：只需对**执行价变动的标的**重新询价——比对新旧 `建议结构.csv` 即可看出哪些变了。

> **反解规则（按 IV 绝对水平，不靠榜单分类；在 `strike_advisor_pipeline.py` 顶部可调）**：
> - IV **30-70%** → 反解出票息落 **15-25%** 的执行价
> - IV **70-100%** → 反解出票息落 **25-30%** 的执行价
> - **执行价硬顶 85%**：中值倒推若 >85% → 改以该档票息下限倒推；仍 >85% → 直接砍到 **85%（敲入 75%）**，票息随行就市（可低于下限）。避免给客户过薄缓冲。
> - IV 太高、票息压不进区间（连 50% 执行价都超上限）→ 默认 **执行价 50 / 敲入 40**
> - 执行价搜索区间 50-95%（`K_MIN/K_MAX`）；硬顶常量 `STRIKE_CAP=85`

> ⚠️ **定期重标定**：建议器的「票息↔IV」关系锁定在 `desk_quotes.csv` 里的历史真实报价上
>（当前 48 点，R²≈0.99）。当交易台报价环境明显变化（利率/借券成本移动，或每周实际
> 询价结果与建议持续偏差）时，把最新一批「执行价 / IV / 真实票息」补进 `desk_quotes.csv`
> 重跑一次，α/β 会自动重标定——留意打印的 R² 是否仍 ≥ 0.95。建议**每月或报价明显漂移时**做一次。

## 出错怎么办

| 现象 | 处理 |
|------|------|
| 窗口显示 ❌ | 截图整个窗口发给 Dave |
| 提示 `未找到 DeepSeek key` | 检查 `deepseek_key.txt` 是否在文件夹里、内容是否只有 key 一行 |
| 提示 `Futu 连接失败` | 不影响生成，想要分析师数据就先打开并登录 OpenD 再重跑 |
| 个别标的分析不完整 | 跑 `python repair_watchlist.py`（自动检测并单独修复），完成后 `git add watchlist.json` → `git commit -m "repair"` → `git push` |
| git push 报错 conflict | 执行 `git pull --rebase` 后重新双击 bat；解决不了找 Dave |

## 只改内容、不重新生成分析（小修改）

打开网站 → 右下角 ADMIN 按钮（密码问 Dave）→ 修改条款/精选 → 导出 watchlist.json →
把下载的文件覆盖到 `fcn-watchlist` 文件夹 → 命令提示符执行：

```
cd %USERPROFILE%\Desktop\fcn-watchlist
git add watchlist.json
git commit -m "manual edit"
git push
```
